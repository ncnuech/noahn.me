    VLC Localization

    As part of Professor Dutta's Lab, I implemented a piece of software which found the relative location of a camera to any known LED lights in the image. The project had a number of steps which follow.
    1. Setup Led light tubes, which blink at specific, unique frequencies.
    2. Start Software side by leveraging openCV to create a bounding rectangle around each "blob" of light in the image(blobs tend to be formed from more than a single light source because of the proximity of the 4 tubes inside of a single box light).
    3. Finding Changes in frequencies through probing the rectangles with FFTs. Using multiple points where the frequency change was over a noise threshold I created a line of best fit between the points minimizing the error while keeping the slope of the rectangle.
    4. With all boxes seperated, the code finds the center of each box(the center of each led tube) and takes the frequency at that point. The frequency is then compared to known position and frequency pairs. This is the end of my section of the code. In the implementation, these matches are passed off to angle of arrival calculations which find location relative to each light located in the image. The process is accurate to the decimeter level. The code can be found on Lab11's Github.
